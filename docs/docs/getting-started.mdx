# Getting Started

This guide will help you get TaskDesk running locally.

## Project Structure

```
taskdesk/
├── landing/      # Marketing site
├── ui/           # Main app (customers)
├── dashboard/    # Admin console
├── server/       # Backend API
└── docs/         # Documentation
```

## Installation

```bash
# Install all dependencies
cd landing && pnpm install
cd ui && pnpm install
cd dashboard && pnpm install
cd server && pnpm install
```

## Database Setup

Ensure PostgreSQL is running and accessible. Create a database:

```bash
createdb taskdesk
```

Copy the example environment file:

```bash
cp server/.env.example server/.env
```

Update `server/.env` with your database URL:

```env
DATABASE_URL=postgresql://user:password@localhost:5432/taskdesk
```

## Development

```bash
# Terminal 1 - Backend (port 8000)
cd server && pnpm dev

# Terminal 2 - Customer app (port 5173)
cd ui && pnpm dev

# Terminal 3 - Dashboard (port 5174)
cd dashboard && pnpm dev

# Terminal 4 - Landing (port 5175)
cd landing && pnpm dev
```

## Access the Applications

| Service | URL |
| --- | --- |
| Customer App | http://localhost:5173 |
| Dashboard | http://localhost:5174 |
| Landing | http://localhost:5175 |
| API | http://localhost:8000 |

## Create Your First Admin User

When the server starts for the first time, it creates a default admin user based on environment variables:

```bash
# In server/.env
ADMIN_SEED_EMAIL=admin@example.com
ADMIN_SEED_PASSWORD=yourpassword
```

Restart the server to create the admin user.

## Environment Variables

### Server (.env)

| Variable | Description | Default |
| --- | --- | --- |
| `PORT` | Server port | `8000` |
| `DATABASE_URL` | PostgreSQL connection string | Required |
| `SESSION_SECRET` | Session cookie secret | Auto-generated |
| `ADMIN_SEED_EMAIL` | Bootstrap admin email | - |
| `ADMIN_SEED_PASSWORD` | Bootstrap admin password | - |
| `SUPPORT_SEED_EMAIL` | Bootstrap support email | - |
| `SUPPORT_SEED_PASSWORD` | Bootstrap support password | - |

### UI (.env)

| Variable | Description | Default |
| --- | --- | --- |
| `VITE_API_URL` | Backend API URL | `http://localhost:8000` |
| `VITE_LIVE_URL` | WebSocket URL | `ws://localhost:8000` |

### Dashboard (.env)

| Variable | Description | Default |
| --- | --- | --- |
| `VITE_API_URL` | Backend API URL | `http://localhost:8000` |
