# API Reference

Base URL: configured via `VITE_API_URL`

## Authentication

### POST /api/auth/register

Create a new account.

**Request:**

```
POST /api/auth/register
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "securepassword"
}
```

**Response:**

```
{
  "user": {
    "id": "...",
    "email": "user@example.com",
    "role": "user",
    "createdAt": 1700000000000
  }
}
```

### POST /api/auth/login

Authenticate and receive session cookie.

**Request:**

```
POST /api/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "securepassword"
}
```

### POST /api/auth/logout

Sign out and clear session.

**Response:**

```
{ "ok": true }
```

### GET /api/auth/me

Get current authenticated user.

**Response:**

```
{
  "user": {
    "id": "...",
    "email": "user@example.com",
    "role": "user"
  }
}
```

## Sessions

### POST /api/sessions

Create a new recording session. Requires authentication.

**Request:**

```
POST /api/sessions
Content-Type: application/json

{
  "url": "https://app.example.com",
  "userAgent": "Mozilla/5.0..."
}
```

**Response:**

```
{ "sessionId": "uuid" }
```

### GET /api/sessions

List sessions. Admin/support only.

**Query Parameters:**

- `page` - Page number (default: 1)
- `limit` - Items per page (default: 20)

### GET /api/sessions/:id/events

Get session events. Requires session ownership or admin.

### POST /api/sessions/:id/events

Append events to session.

### DELETE /api/sessions/:id

Delete session. Requires admin.

## Tasks

### GET /api/tasks

List tasks for authenticated user.

Each authenticated user only sees their own tasks.

### POST /api/tasks

Create a new task.

### PUT /api/tasks/:id

Update task.

### DELETE /api/tasks/:id

Delete task.

## Users (Admin Only)

### GET /api/users

List users with pagination.

### POST /api/users

Create a new user (admin only).

### PUT /api/users/:id

Update user.

### DELETE /api/users/:id

Delete user.

## WebSocket

### /live

Live replay WebSocket endpoint.

**Query Parameters:**

- `session` - Session ID
- `role` - Either `recorder` or `viewer`

**Recorder Role:**

- Sends events in real-time
- Authenticates via session ownership

**Viewer Role:**

- Receives events in real-time
- Can view live playback
