# Production Guide

## Environment Variables

| Variable | Default | Description |
|----------|----------|-------------|
| `DATABASE_URL` | `postgres://postgres:postgres@localhost:5434/taskdesk` | PostgreSQL connection string |
| `PORT` | `8000` | Server port |

## Database

The application uses PostgreSQL. The schema is created automatically on server startup:

```sql
CREATE TABLE sessions (
  id TEXT PRIMARY KEY,
  url TEXT DEFAULT 'unknown',
  user_agent TEXT DEFAULT '',
  started_at TIMESTAMP NOT NULL DEFAULT NOW(),
  last_event_at TIMESTAMP NOT NULL DEFAULT NOW(),
  event_count INTEGER DEFAULT 0
);

CREATE TABLE session_events (
  id SERIAL PRIMARY KEY,
  session_id TEXT NOT NULL,
  events TEXT NOT NULL,
  created_at TIMESTAMP NOT NULL DEFAULT NOW(),
  FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE
);

CREATE TABLE tasks (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  description TEXT DEFAULT '',
  status TEXT NOT NULL DEFAULT 'todo',
  created_at TIMESTAMP NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMP NOT NULL DEFAULT NOW()
);
```

### PostgreSQL

```js
// Store events
await pool.query(
  'INSERT INTO session_events (session_id, events) VALUES ($1, $2)',
  [sessionId, JSON.stringify(events)]
);

// Fetch events
const rows = await pool.query(
  'SELECT events FROM session_events WHERE session_id = $1 ORDER BY id',
  [sessionId]
);
const allEvents = rows.rows.flatMap(r => r.events);
```

### S3 for Large Sessions

```js
import { S3Client, PutObjectCommand } from '@aws-sdk/client-s3';

await s3.send(new PutObjectCommand({
  Bucket: 'my-taskdesk-sessions',
  Key: `sessions/${sessionId}.json.gz`,
  Body: gzip(JSON.stringify(events)),
  ContentEncoding: 'gzip',
}));
```

## Sampling

Record a percentage of sessions:

```js
const SAMPLE_RATE = 0.1; // 10% of visitors
const shouldRecord = Math.random() < SAMPLE_RATE;
```

## WebSocket Security

- Use `wss://` (WebSocket Secure) in production
- Ensure your proxy supports WebSocket upgrades
- Consider adding authentication middleware
